## Venues

**venues**

* id
* name
* address
* city
* capacity (nullable)
* description (nullable)
* image (nullable)
* status (active/inactive)
* created_at
* updated_at

---

## Events

**events**

* id
* title
* date
* time
* venue_id (foreign key → venues.id)
* description
* image (nullable)
* status (upcoming/completed/cancelled)
* created_at
* updated_at

---

## Event Management (Categories – optional but recommended)

**event_categories**

* id
* name
* description (nullable)
* status
* created_at
* updated_at

**event_category_map** (if one event can have multiple categories)

* id
* event_id (foreign key → events.id)
* category_id (foreign key → event_categories.id)

---

## User Management

**users**

* id
* name
* email
* password
* phone (nullable)
* role (admin/user)
* status (active/blocked)
* created_at
* updated_at

---

## Bookings / Registrations

**bookings**

* id
* user_id (foreign key → users.id)
* event_id (foreign key → events.id)
* booking_date
* number_of_tickets
* status (pending/approved/cancelled)
* created_at
* updated_at

---

## Payments (only if events are paid)

**payments**

* id
* booking_id (foreign key → bookings.id)
* amount
* payment_method (cash/bkash/nagad/card)
* transaction_id (nullable)
* payment_status (pending/paid/failed)
* payment_date
* created_at
* updated_at

---

## Admin (optional – if you separate admins)

**admins**

* id

* password
* created_at
* updated_at

**settings**

* id
